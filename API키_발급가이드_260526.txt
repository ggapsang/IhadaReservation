================================================================
이하다 예약 시스템 - API 키 발급 가이드 (알리고 기준)
작성일: 2026-05-26
대상: 고객사 담당자 (비개발자도 따라 가능)
================================================================

[먼저 읽어주세요 - 가장 헷갈리시는 부분]

"결제 키"와 "알림톡 키"는 완전히 다른 서비스의 키입니다.
같은 화면, 같은 사이트에서 발급되지 않습니다.

  ┌──────────────────────┬─────────────────────────────┐
  │ 키 종류              │ 어디서 받나요?              │
  ├──────────────────────┼─────────────────────────────┤
  │ 결제 키 (3종)        │ 토스페이먼츠 사이트         │
  │                      │ tosspayments.com            │
  │                      │ -> 개발자센터 -> API 키     │
  ├──────────────────────┼─────────────────────────────┤
  │ 알림톡 키 (2종)      │ 알리고 사이트               │
  │                      │ smartsms.aligo.in           │
  │                      │ -> 알림톡 -> API 키 관리    │
  └──────────────────────┴─────────────────────────────┘

즉, 지금 알리고 사이트에 보이는 API KEY는 "알림톡 발송용"이고,
결제 키는 토스페이먼츠 사이트에서 따로 발급받으셔야 합니다.

================================================================
[이 시스템이 사용하는 외부 서비스]
================================================================

이 시스템은 외부 서비스 4종(토스페이먼츠, 카카오 알림톡(알리고 경유),
구글 캘린더, 구글 드라이브)과 연결됩니다. 외부 서비스마다
"우리는 누구인지"를 증명하는 비밀번호 같은 값이 필요한데,
이것을 "API 키"라고 부릅니다.

키가 여러 개로 보이는 이유는 두 가지입니다.
  (1) 서비스마다 사용처가 다르기 때문 (결제용, 알림용 등)
  (2) 한 서비스 안에서도 용도별로 분리됨
      예: 토스는 "결제창 띄우기용"과 "서버 통신용"을 분리

키는 비밀번호와 같습니다. 절대 카카오톡, 이메일 본문, 채팅으로
공유하지 말고, 반드시 안전한 방법(전화로 불러주기, 또는
시스템 관리자가 직접 입력)으로 전달해야 합니다.

================================================================
[전체 키 목록 한눈에]
================================================================

총 12개. 그러나 고객사가 직접 발급받아야 하는 것은 8개입니다.

A. 결제 관련 (3개) - 토스페이먼츠에서 발급
   1. PAYMENT_CLIENT_KEY      결제창 띄우기용 (공개 가능)
   2. PAYMENT_SECRET_KEY      서버 통신용 (비공개)
   3. PAYMENT_WEBHOOK_SECRET  위조 방지 검증용 (비공개)

B. 알림톡 관련 (2개) - 알리고에서 발급
   4. KAKAO_API_KEY           알리고 발송 API 인증용
   5. KAKAO_SENDER_KEY        카카오톡 채널(발신자) 식별용 (알리고에 등록)

C. 관리자 정보 (2개) - 고객사 내부에서 결정
   6. ADMIN_EMAIL             예약 알림 받을 이메일
   7. ADMIN_PHONE             관리자 연락처

D. 구글 자원 ID (3개) - 시스템 관리자가 작업
   8. CALENDAR_ID_CONFIRMED   확정 예약 표시할 구글 캘린더 ID
   9. UPLOAD_FOLDER_ID        사업자등록증 파일이 저장될 폴더 ID
   10. TERMS_DOC_ID           약관 문서 ID

E. 기타 (2개) - 미사용 또는 자동 설정
   11. CALENDAR_ID_PENDING    (Phase 2 이후 사용 예정)
   12. PAYMENT_PROVIDER       (자동으로 "tosspayments" 고정)

================================================================
[그룹 A] 토스페이먼츠 - 결제 키 3종 발급
================================================================

>>> 지금 알리고 사이트에서 보고 계신 키와는 무관한 별도 작업입니다.
>>> 결제를 받으려면 반드시 이 그룹의 키 3개가 필요합니다.

[사전 준비물]
  - 사업자등록증 사본 (PDF)
  - 대표자 신분증 사본
  - 정산받을 통장 사본 (대표자 또는 법인 명의)
  - 통신판매업 신고증 (인터넷 결제 시 의무, 관할 구청에서 발급)
  - 사업장 대표 전화번호와 이메일

[발급 절차]

1단계. 토스페이먼츠 가맹점 가입
  - 웹사이트: https://www.tosspayments.com
  - 우측 상단 "가입하기" 클릭
  - 사업자 정보 입력 + 위 서류 업로드
  - 심사 소요: 영업일 기준 2~5일

2단계. 심사 통과 후 개발자센터 진입
  - 로그인 후 우측 상단 "개발자센터" 메뉴
  - 좌측 메뉴 "API 키" 클릭

3단계. 키 3개 확인 및 전달

  (1) PAYMENT_CLIENT_KEY (클라이언트 키)
      - "API 키" 화면 상단에 표시됨
      - "test_ck_" 또는 "live_ck_" 로 시작
      - 처음에는 테스트 키 사용, 실서비스 전환 시 라이브 키로 교체

  (2) PAYMENT_SECRET_KEY (시크릿 키)
      - 같은 화면 클라이언트 키 아래에 표시됨
      - "test_sk_" 또는 "live_sk_" 로 시작
      - !!! 중요: 이 값은 절대 외부 공개 금지

  (3) PAYMENT_WEBHOOK_SECRET (웹훅 시크릿)
      - 좌측 메뉴 "웹훅(Webhook)" 클릭
      - "엔드포인트 추가" 버튼
      - URL 입력란에는 시스템 관리자가 알려주는 주소를 넣음
        (예: https://script.google.com/macros/s/.../exec)
      - 저장하면 화면에 "시그니처 시크릿" 값이 표시됨
      - 이 값을 복사 (한 번만 표시되므로 캡처 권장)

[테스트 키 vs 라이브 키]
  - 테스트 키: 실제 결제 안 됨, 테스트용 카드번호로만 결제 시뮬레이션
  - 라이브 키: 실제 카드 결제 발생, 실제 정산됨
  - 시스템 점검 단계에서는 테스트 키로 충분합니다.
  - 실서비스 오픈일에 라이브 키로 교체합니다.

[비용]
  - 가입비 없음
  - 결제 건당 수수료 (카드 약 2.5~3%, 계좌이체 약 1.5%)
  - 정산 주기 협의 가능 (보통 D+3)

================================================================
[그룹 B] 알림톡 - 알리고 키 2종 발급
================================================================

>>> 알리고 가입은 이미 완료하셨고, 현재 사업자 인증 심사 중입니다.
>>> 심사 통과(영업일 기준 1~3일) 후 아래 단계를 이어서 진행하시면 됩니다.

알림톡은 카카오 단독으로는 발송할 수 없고, 카카오와 계약된
"비즈메시지 중계 솔루션사"를 통해서만 발송됩니다.
본 시스템은 "알리고(aligo.in)"를 사용합니다.

전체 작업 흐름:
  단계 1: 카카오톡 채널 개설          (카카오 비즈니스에서) [선행 필요]
  단계 2: 비즈니스 채널로 인증 승급    (카카오 비즈니스에서) [선행 필요]
  단계 3: 알리고 가입                  [완료]
  단계 4: 알리고 사업자 인증           [심사 중]
  단계 5: 알리고에 카카오 채널 연결    [심사 통과 후]
  단계 6: API 키 2종 확인              [그 다음]
  단계 7: 알림톡 템플릿 등록 + 심사

[1단계: 카카오톡 채널 개설] (별도 진행 - 이미 있으시면 생략)

  - 웹사이트: https://business.kakao.com
  - 카카오 계정으로 로그인
  - "카카오톡 채널" 메뉴 -> "새 채널 만들기"
  - 채널 이름: "이하다" 같은 브랜드명
  - 검색용 아이디 설정 (예: @ihada)
  - 즉시 개설됨

[2단계: 카카오 비즈니스 인증]

  - 같은 카카오 비즈니스 사이트에서 진행
  - 개설한 채널 관리 화면 -> "비즈니스 채널 전환" 메뉴
  - 사업자등록증 업로드 + 사업자 정보 입력
  - 심사 소요: 영업일 기준 1~3일
  - 인증 완료 시 채널명 옆에 인증 마크 표시
  - 이 단계 완료 후에만 알림톡 발송 자격이 부여됩니다.

[3~4단계: 알리고 가입 + 사업자 인증]  >>> 현재 여기 진행 중

  - 알리고 회원가입 완료 (체크)
  - 사업자 인증 신청 완료, 심사 중 (체크)
  - 심사 소요: 영업일 기준 1~3일
    (주말/공휴일에는 진행 안 됨 - 평일에 결과 확인)

[5단계: 알리고에 카카오 채널 연결 (발신프로필 등록)]

  알리고 사업자 인증이 통과된 후에 진행합니다.

  - 알리고 로그인: https://smartsms.aligo.in
  - 좌측 메뉴 "알림톡" -> "발신프로필 관리" -> "발신프로필 등록"
  - 1~2단계에서 만든 카카오톡 채널의 "검색용 아이디" 입력 (예: @ihada)
  - 카테고리 선택, 채널 관리자 카카오톡으로 인증번호 수신 후 입력
  - 카카오 측 자동 검증 (1~2 영업일)
  - 검증 완료되면 발신프로필 목록에 활성 상태로 표시됨

[6단계: 발급되는 키 2개 확인]

  알리고는 두 키를 각각 다른 메뉴에서 확인합니다.

  (4) KAKAO_API_KEY
      - 알리고가 우리 시스템을 식별하는 API 키
      - 위치: 알리고 로그인 -> 우측 상단 "내 정보" 또는
              좌측 "API 연동" -> "API Key 발급/확인"
      - 형식: 영문+숫자 조합 (보통 20~30자리)
      - "API Key 발급" 버튼을 처음 누르면 생성됨

  (5) KAKAO_SENDER_KEY (발신프로필 키)
      - 우리 카카오톡 채널을 식별하는 키
      - 위치: 알리고 로그인 -> "알림톡" -> "발신프로필 관리"
              -> 등록된 발신프로필의 "발신프로필 키" 컬럼
      - 형식: 영문+숫자 조합

  >>> 이 두 값을 복사해서 시스템 관리자에게 전달

[7단계: 알림톡 템플릿 등록 (필수)]

  알림톡은 자유 문구를 보낼 수 없고, 카카오가 미리 심사한
  "템플릿"만 발송 가능합니다. 발송 종류별로 템플릿 등록이 필요합니다.

  필요한 템플릿 예시:
    - 예약 접수 안내
    - 결제 완료 안내
    - 예약 확정 안내
    - 예약 1일 전 리마인드
    - 예약 취소/환불 안내

  등록 위치: 알리고 -> "알림톡" -> "템플릿 관리" -> "템플릿 등록"
  심사 소요: 영업일 기준 2~5일 (카카오 측 심사)
  >>> 발신프로필 등록과 동시에 시작해야 전체 일정이 짧아집니다.
  >>> 템플릿 문구 초안은 시스템 관리자가 별도로 제공합니다.

[알리고 비용 안내]
  - 가입비/월정액 없음
  - 선불 충전 방식 (충전한 만큼 사용)
  - 알림톡 건당 약 9~13원
  - 월 예약 500건 기준 약 5천~7천원 예상
  - 첫 충전은 1만원부터 가능

================================================================
[그룹 C] 관리자 정보 - 직접 결정
================================================================

  (6) ADMIN_EMAIL
      - 새 예약이 들어오면 알림 받을 이메일
      - 1개만 가능 (복수 필요 시 그룹 이메일 또는 별도 협의)
      - 예: reservation@ihada.kr

  (7) ADMIN_PHONE
      - 비상 연락처
      - 형식: 010-0000-0000
      - SMS 발송용으로 쓰일 수 있음

================================================================
[그룹 D] 구글 자원 ID - 시스템 관리자 작업
================================================================

이 3개는 고객사가 직접 발급할 필요는 없으나, 어떤 캘린더에
예약을 표시할지 등은 결정해주셔야 합니다.

  (8) CALENDAR_ID_CONFIRMED
      - 확정된 예약이 표시될 구글 캘린더
      - 선택지:
        (가) 기존 캘린더 사용 (값 비워두면 됨)
        (나) "이하다 예약" 전용 캘린더를 새로 만들어 사용
      - 시스템 관리자가 추출하여 입력

  (9) UPLOAD_FOLDER_ID
      - 고객이 업로드한 사업자등록증이 저장될 폴더
      - 자동으로 "예약_사업자등록증" 폴더가 생성됨
      - 별도 폴더 지정 원하시면 알려주세요

  (10) TERMS_DOC_ID
      - 약관 본문이 담긴 구글 문서
      - 기존 문서를 그대로 사용 중 (변경 필요 시만 신규 ID 전달)

================================================================
[전달 방법 - 보안 주의]
================================================================

키를 시스템 관리자에게 전달할 때:

  좋은 방법
    - 전화로 직접 불러주기
    - 1Password, Bitwarden 등 비밀번호 매니저 공유
    - 만료 시한이 있는 메모 서비스 (예: privnote.com)

  나쁜 방법 (절대 금지)
    - 카카오톡, 이메일 본문에 그대로 적기
    - 슬랙 일반 채널에 붙여넣기
    - 회사 공용 노트(Notion 등)에 평문으로 기록

  >>> 만약 실수로 위 채널에 노출되었다면, 즉시 해당 키를
      "재발급(rotation)" 해야 합니다.
      각 서비스 대시보드에서 "재발급" 또는 "삭제 후 재생성"
      메뉴로 새 값을 받을 수 있습니다.

================================================================
[리드타임 정리 - 발급 일정 계획]
================================================================

각 작업의 소요 시간을 고려해 역산해주세요.

  카카오톡 채널 개설           : 즉시
  카카오 비즈니스 인증         : 1~3 영업일
  알리고 가입                  : 완료
  알리고 사업자 인증           : 1~3 영업일 (심사 중)
  알리고 발신프로필 카카오 검증: 1~2 영업일
  알림톡 템플릿 카카오 심사    : 2~5 영업일 (반려 시 재신청)
  토스페이먼츠 가맹점 심사     : 2~5 영업일
  토스 웹훅 등록               : 즉시

  >>> 가장 오래 걸리는 것: 알림톡 템플릿 심사 (최대 1~2주)
  >>> 알리고 사업자 인증 통과 직후 바로 "발신프로필 등록"과
      "템플릿 등록"을 동시에 신청하시면 일정을 최대 절반으로

Q1. 알리고에 보이는 API KEY가 결제 키인가요?
  - 아니요. 알리고의 API KEY는 "알림톡 발송용"입니다.
  - 결제 키는 토스페이먼츠 사이트(tosspayments.com)에서 별도로
    발급받아야 합니다. 두 서비스는 완전히 분리되어 있습니다.

Q2. 키 발급에 비용이 드나요?
  - 토스, 카카오 채널 개설: 무료
  - 알리고: 가입비 무료, 발송 건당 과금 (선불 충전)
  - 통신판매업 신고: 지자체별 약 4만원 내외

Q3. 테스트 단계에서도 모든 키가 필요한가요?
  - 아니오. 결제 테스트 키 + 관리자 이메일만 있어도 핵심 기능 점검 가능.
  - 알림톡, 라이브 결제 키는 실서비스 직전에 채워도 됩니다.

Q4. 키를 잃어버리면?
  - 시크릿 키, 웹훅 시크릿은 재발급만 가능 (조회 불가).
  - 알리고 API 키는 재발급 시 기존 키 즉시 무효화됩니다.
  - 재발급 시 시스템 관리자에게 즉시 새 값 전달 필요.

Q5. 사업자 명의 변경이 있으면?
  - 토스: 가맹점 정보 변경 절차 필요 (서류 재제출)
  - 카카오/알리고: 채널 및 알리고 사업자 인증 재진행
  - 두 경우 모두 사전에 시스템 관리자와 일정 협의 필요.

Q6. 알리고 말고 다른 솔루션사를 쓰고 싶으면?
  - 가능하지만 코드 수정이 필요합니다 (API 호출 방식이 회사마다 다름).
  - 본 가이드는 알리고 기준입니다.

Q7. 토스 말고 다른 PG사를 쓰고 싶으면?
  - 가능하지만 코드 수정이 필요합니다.
  - 본 가이드는 토스페이먼츠 기준입니다.
