## 1. 파일별 함수 매핑

### `10_Config.gs` — 설정 & Properties
| 함수 | 용도 |
|---|---|
| `getSettings` | 설정 시트 전체 조회 (클라이언트 호출). |

## 4. 초기 세팅 — `setupAll()` 1회 실행 (2분)

⚠️ **이 단계에서 기존 시트 데이터가 삭제됩니다.** 백업이 끝났는지 확인 후 진행하십시오.

1. Apps Script 에디터 상단 함수 드롭다운에서 **`setupAll`** 선택.
2. ▶ `실행`.
3. 권한 승인 팝업이 처음 뜨면 본인 계정으로 승인.
4. 경고창이 표시됩니다:
   ```
   ⚠️ 초기 세팅
   1) 예약내역·예약현황로그의 모든 데이터를 삭제합니다.
   2) 레거시 시트(Room정보·옵션상품·임시주문·결제로그)를 통째로 삭제합니다.
   3) 설정 시트의 옛 키를 모두 제거하고 명세 기준 키만 남깁니다.
   4) Properties를 빈 값으로 초기화합니다.
   5) 예약내역 25컬럼 헤더를 새로 작성합니다.
   6) 명세 기준 가격·정책을 설정 시트에 작성합니다.
   계속하시겠습니까?
   ```
   - **[예]**: 위 4단계를 모두 수행.
   - **[아니요]**: 즉시 중단 (`{aborted: true}` 반환). 데이터 보존하며 누락분만 보강하려면 `setupAll_keepData()`를 대신 실행.
5. 완료 알림창이 표시되면 OK.

### 결과 확인
스프레드시트 탭에서 직접 확인합니다.

| 가격이 모두 0원 | 설정 시트 누락 또는 `migrateSettings_260522` 미실행 | `setupAll_keepData()` 실행하여 누락분 보강 |
| 평일·주말 가격이 같음 | 폼에서 날짜 미선택 | 정상 동작 (날짜 선택 시 즉시 갱신됨) |
| 이메일 입력칸이 없음 | index.html 구 버전 | index.html 최신 내용으로 재배포 |
| 함수 드롭다운에 `suggestRoomType` 등 옛 함수가 보임 | 구 `.gs` 파일이 남아있음 | 좌측 파일 목록에서 10개 외 파일 삭제 |
| Calendar 이벤트 생성 실패 | `CALENDAR_ID_CONFIRMED` 잘못된 ID | Property 비우면 기본 Calendar 사용 / 올바른 ID 확인 |
| 약관 모달 비어있음 | `TERMS_DOC_ID` 잘못된 ID | Property 비우면 폴백 작동 / URL에서 ID 재추출 |
